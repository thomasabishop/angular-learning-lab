Learning testing

The sub-components in this project all exist in order for me to learn and practise unit testing in Angular.

Testing a service

When you enter some text in the input field below and click enter, an alert is triggered which returns the character length of the text that has been entered. The character length calculation is handled by a service CountCharsService.

Testing components

The tests applied in this section all focus on testing a component. There are three variations:

  • Testing the component class-only without dependencies
  • Testing the component class-only with dependencies
  • Testing the component with DOM-specific controls

Testing component class-only

Here the unit test simply confirms that the property which holds the mood value begins as 'happy' and then changes to 'sad' after the click event. This is achieved purely by calling the method that associated with the click. There is no engagement with the UI or the DOM itself.

Testing component class-only with dependencies

Here we call the date via a service that executes within the ngOnInit lifecycle method and outputs using a pipe. This is so that we can create a class test which requires a service. As the date will always be different, we use a test rule that checks the date format against a regex string.
